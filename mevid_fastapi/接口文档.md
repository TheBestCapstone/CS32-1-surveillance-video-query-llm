# MEVID FastAPI 接口文档

**版本**: 1.0.0  
**服务描述**: 多摄像头视频管道（YOLO + Re-ID + 跨摄像头匹配）与 LangGraph Agent 问答管道的统一 HTTP 服务。  
**启动命令**:
```bash
cd mevid_fastapi
uvicorn main:app --port 8765
```
**交互式文档**: 启动后访问 `http://127.0.0.1:8765/docs`

---

## 接口列表

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/healthz` | 健康检查，返回当前数据库状态 |
| POST | `/api/v1/ingest/seeds` | 从预生成的 vector flat 文件构建 SQLite + Chroma |
| POST | `/api/v1/ingest/videos` | 从原始视频跑完整多摄像头 pipeline |
| POST | `/api/v1/database/select` | 手动切换已有数据库 |
| POST | `/api/v1/agent/query` | 单条自然语言查询 |
| POST | `/api/v1/eval/run` | 跑完整评测套件，返回准确率/IoU 指标 |

---

## 标准调用流程

```
启动服务
  ↓
POST /api/v1/ingest/seeds   （或 /ingest/videos）
  ↓ 自动切换数据库
POST /api/v1/agent/query    （逐条查询）
  ↓
POST /api/v1/eval/run       （批量评测）
```

## 接口详情

### GET `/healthz`

健康检查，返回服务状态和当前选中的数据库信息。

**响应示例**

```json
{
  "status": "ok",
  "database_selected": true,
  "sqlite_path": "/path/to/episodic_events.sqlite",
  "chroma_path": "/path/to/chroma",
  "namespace": "mevid_eval"
}
```

---

### POST `/api/v1/ingest/seeds`

从预生成的 `*_events_vector_flat.json` 种子文件构建 SQLite + Chroma 数据库，**并自动将 Agent 切换到新建的数据库**。

适用于已完成视频处理（Pipeline Phase 1–5）、只需重建检索库的场景。

**请求体**

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `seeds_dir` | string | ✅ | — | `*_events_vector_flat.json` 文件所在目录（绝对路径） |
| `namespace` | string | | `"mevid"` | Chroma 集合命名空间前缀 |
| `reset_existing` | bool | | `true` | 是否重建已有数据库 |

**请求示例**

```bash
curl -X POST http://127.0.0.1:8765/api/v1/ingest/seeds \
  -H "Content-Type: application/json" \
  -d '{
    "seeds_dir": "/path/to/events_vector_flat",
    "namespace": "mevid_eval",
    "reset_existing": true
  }'
```

**响应示例**

```json
{
  "status": "ok",
  "namespace": "mevid_eval",
  "sqlite_path": "/path/mevid_fastapi/data/runtime/mevid_eval/episodic_events.sqlite",
  "chroma_path": "/path/mevid_fastapi/data/runtime/mevid_eval/chroma",
  "seed_file_count": 6,
  "sqlite_rows": 697,
  "chroma_child_records": 620,
  "chroma_ge_records": 5,
  "elapsed_ms": 85000.0
}
```

**Chroma 集合命名规则**

构建完成后自动创建以下集合（与 `run_agent_eval.py` 命名完全一致）：

| 集合 | 名称 | 说明 |
|------|------|------|
| 子集合（child） | `{namespace}_tracks` | Track 级语义检索 |
| 父集合（parent） | `{namespace}_tracks_parent` | 10 分钟时间窗上下文 |
| 事件集合（event） | `{namespace}_events` | 事件级精细检索 |
| 全局实体（GE） | `mul_camera_global_entities` | 跨摄像头实体（固定名） |

### POST `/api/v1/ingest/videos`

从原始视频文件跑完整多摄像头 Pipeline（YOLO 检测 → OSNet Re-ID → 拓扑学习 → 跨摄像头匹配），生成种子文件并构建数据库，**并自动切换 Agent 到新数据库**。

> 注意：此接口会对每路视频跑 YOLO 推理，5 分钟 × 6 路摄像头约需 30–60 分钟（CPU）。

**请求体**

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `camera_videos` | dict[str, str] | ✅ | — | `{摄像头ID: 视频绝对路径}` |
| `namespace` | string | | `"mevid"` | Chroma 命名空间前缀 |
| `conf` | float | | `0.40` | YOLO 检测置信度阈值 |
| `iou` | float | | `0.40` | YOLO 追踪 IoU 阈值 |
| `reid_device` | string | | `"cpu"` | Re-ID 运行设备（`"cpu"` 或 `"cuda"`） |
| `refine` | bool | | `false` | 是否调用 LLM 事件细化（Phase 5） |
| `refine_model` | string | | `"qwen-vl-max"` | LLM 细化模型名 |
| `reset_existing` | bool | | `true` | 是否重建已有数据库 |

**请求示例**

```bash
curl -X POST http://127.0.0.1:8765/api/v1/ingest/videos \
  -H "Content-Type: application/json" \
  -d '{
    "camera_videos": {
      "G329": "/data/mevid_slots/13-50/G329.avi",
      "G421": "/data/mevid_slots/13-50/G421.avi",
      "G424": "/data/mevid_slots/13-50/G424.avi",
      "G506": "/data/mevid_slots/13-50/G506.avi",
      "G508": "/data/mevid_slots/13-50/G508.avi",
      "G328": "/data/mevid_slots/13-50/G328.avi"
    },
    "namespace": "slot_13_50",
    "conf": 0.40,
    "iou": 0.40,
    "reid_device": "cpu",
    "refine": false
  }'
```

**响应结构** 与 `/api/v1/ingest/seeds` 相同，额外包含 `pipeline_cache_path` 字段。

### POST `/api/v1/database/select`

手动将 Agent 切换到已有的 SQLite + Chroma 数据库（不重建），适用于复用上次构建结果。

**请求体**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `namespace` | string | ✅ | Chroma 命名空间前缀 |
| `sqlite_path` | string | ✅ | SQLite 数据库绝对路径 |
| `chroma_path` | string | ✅ | Chroma 目录绝对路径 |

**请求示例**

```bash
curl -X POST http://127.0.0.1:8765/api/v1/database/select \
  -H "Content-Type: application/json" \
  -d '{
    "namespace": "mevid_eval",
    "sqlite_path": "/path/to/episodic_events.sqlite",
    "chroma_path": "/path/to/chroma"
  }'
```

**响应示例**

```json
{ "status": "ok", "namespace": "mevid_eval" }
```

### POST `/api/v1/agent/query`

向 LangGraph Agent 发送单条自然语言问题，返回答案和检索上下文。调用前必须已选择数据库。

**Agent 节点链**

```
self_query_node → query_classification_node → parallel_retrieval_fusion_node
→ match_verifier_node → final_answer_node → summary_node
```

**请求体**

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `query` | string | ✅ | — | 自然语言问题 |
| `thread_id` | string\|null | | `null` | 对话线程 ID（null 则自动生成） |
| `user_id` | string | | `"mevid_user"` | 用户标识 |
| `include_rows` | bool | | `false` | 是否在响应中包含检索到的原始行 |
| `include_node_trace` | bool | | `true` | 是否返回节点执行路径 |
| `top_k_rows` | int | | `5` | 返回的最大检索行数 |

**请求示例**

```bash
curl -X POST http://127.0.0.1:8765/api/v1/agent/query \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Did a person with dark coat appear in camera G329 and then appear again in camera G421?",
    "include_rows": true,
    "include_node_trace": true
  }'
```

**响应示例**

```json
{
  "query": "Did a person with dark coat appear in camera G329 and then appear again in camera G421?",
  "answer": "Yes, a person with a dark coat was tracked across camera G329 and G421 (person_global_3, 50.4s–122.8s).",
  "answer_type": "cross_camera",
  "node_trace": [
    "self_query_node", "query_classification_node",
    "parallel_retrieval_fusion_node", "match_verifier_node",
    "final_answer_node", "summary_node"
  ],
  "elapsed_ms": 7200.0,
  "verifier_result": {
    "verdict": "exact",
    "entity_id": "person_global_3",
    "matched_cameras": ["G329", "G421"]
  },
  "rows": [
    {
      "video_id": "2018-03-11.13-50-01.13-55-01.school.G421.r13",
      "track_id": "person_global_3",
      "start_time": 50.4,
      "end_time": 63.3,
      "event_text_en": "person_global_3 walked across..."
    }
  ]
}
```

**错误响应**

| HTTP 状态码 | 原因 |
|------------|------|
| 400 | query 为空，或尚未选择数据库 |
| 500 | Agent 内部错误 |

### POST `/api/v1/eval/run`

对 `sampled_10.json` 中全部问题跑一遍 Agent 评测，返回准确率、Top-Hit 率、IoU 等汇总指标。调用前必须已选择数据库。

> 10 题全量评测约需 2–3 分钟（HTTP 超时请设为 600s 以上）。

**请求体**

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `questions_file` | string\|null | | `null` | 问题集 JSON 文件路径；null 则使用内置 `sampled_10.json` |

**请求示例**

```bash
curl -X POST http://127.0.0.1:8765/api/v1/eval/run \
  -H "Content-Type: application/json" \
  -d '{}' \
  --max-time 600
```

**响应示例（汇总部分）**

```json
{
  "total": 10,
  "correct": 7,
  "accuracy": 0.7,
  "top_hit_total": 8,
  "top_hit_rate": 0.8,
  "mean_iou": 0.250,
  "iou_threshold": 0.15,
  "n_iou_pass": 6,
  "iou_pass_rate": 0.6,
  "individual_iou": [0.177, 0.413, 0.223, 0.395, 0.069, 0.298, 0.0, 0.426],
  "by_category": {
    "appearance":   { "total": 3, "correct": 3, "accuracy": 1.0, "top_hit": 3, "top_hit_rate": 1.0 },
    "event":        { "total": 2, "correct": 2, "accuracy": 1.0, "top_hit": 2, "top_hit_rate": 1.0 },
    "cross_camera": { "total": 3, "correct": 2, "accuracy": 0.67, "top_hit": 1, "top_hit_rate": 0.33 },
    "negative":     { "total": 2, "correct": 0, "accuracy": 0.0, "top_hit": 2, "top_hit_rate": 1.0 }
  },
  "cases": [ ... ],
  "elapsed_total_ms": 162000.0
}
```

**IoU 说明**

- 仅对预期答案为 `yes` 的问题计算 IoU
- 预测时间范围优先从 Agent 最终回答文本中提取（格式 `M:SS-M:SS` 或 `Xs-Xs`），其次从检索行的 `start_time/end_time` 字段取
- 计算时对预测范围施加 ±30s padding 以容纳检索粒度误差
- `iou_pass = correct AND iou >= 0.15`（与 `run_agent_eval.py` 完全一致）

## 测试结果

### 评测配置

| 配置项 | 值 |
|--------|-----|
| 数据集 | MEVID slot 13-50，6 路摄像头 |
| 问题数量 | 10 题（sampled_10.json） |
| 检索层级 | child（track 级，`AGENT_CHROMA_RETRIEVAL_LEVEL=child`） |
| Parent Projection | 关闭 |
| 执行模式 | parallel_fusion（SQL + Hybrid 并行融合） |

### 最终指标

| 指标 | 结果 |
|------|------|
| **答案准确率** | **7/10 = 70.0%** |
| **Top-Hit 率** | **8/10 = 80.0%** |
| Mean IoU（yes 题） | 0.250 |
| IoU ≥ 0.15 通过率 | 6/10 = 60.0% |

### 按类别

| 类别 | 题数 | 答案准确率 | Top-Hit |
|------|------|-----------|---------|
| appearance | 3 | 3/3 (100%) | 3/3 (100%) |
| event | 2 | 2/2 (100%) | 2/2 (100%) |
| cross_camera | 3 | 2/3 (67%) | 1/3 (33%) |
| negative | 2 | 0/2 (0%) | 2/2 (100%) |

### 逐题结果

| # | 类别 | 摄像头 | 预期 | 预测 | Top-Hit | IoU |
|---|------|--------|------|------|---------|-----|
| Q01 | appearance | G329 | yes | ✅ yes | ✅ | 0.177 |
| Q02 | appearance | G508 | yes | ✅ yes | ✅ | 0.413 |
| Q03 | appearance | G421 | yes | ✅ yes | ✅ | 0.223 |
| Q04 | event | G508 | yes | ✅ yes | ✅ | 0.395 |
| Q05 | event | G328 | yes | ✅ yes | ✅ | 0.069 |
| Q06 | cross_camera | G328→G424 | yes | ✅ yes | ❌ | 0.298 |
| Q07 | cross_camera | G424→G506 | yes | ❌ no | ❌ | 0.000 |
| Q08 | negative | G421→G508 | no | ❌ yes | ✅ | — |
| Q09 | cross_camera | G329→G421 | yes | ✅ yes | ✅ | 0.426 |
| Q10 | negative | G328→G421 | no | ❌ yes | ✅ | — |

### 错误分析

**Q7（cross_camera 误判）**: Pipeline 追踪结果中 `person_global_2`（浅灰色卫衣）仅覆盖 G329/G328/G421，GE 集合中无 G424→G506 记录，属于 pipeline 追踪数据与 ground truth 不一致的数据层面问题。

**Q8/Q10（negative 误判）**: "dark hoodie" / "dark long coat" 描述与数据库中 "dark clothing" 语义高度重叠，verifier 基于摄像头覆盖和外观描述给出了误判。负样本检测需要更高精度的外观区分能力。

## 端到端调用脚本

`run_mevid_eval.py` 封装了完整的三步流程：

```bash
# 1. 启动服务（Terminal 1）
cd mevid_fastapi
uvicorn main:app --port 8765

# 2. 跑端到端评测（Terminal 2）
python mevid_fastapi/run_mevid_eval.py
```

脚本依次调用：
1. `GET /healthz` — 确认服务就绪
2. `POST /api/v1/ingest/seeds` — 构建数据库（使用 `events_vector_flat/` 种子）
3. `POST /api/v1/eval/run` — 跑 10 题评测，打印汇总表，保存 JSON 到 `data/runtime/{namespace}/fastapi_eval_result.json`

## 文件结构

```
mevid_fastapi/
├── main.py              # FastAPI 应用入口，路由定义

├── pipeline_service.py  # 视频管道服务（seeds 导入 + 多摄像头 pipeline）
├── eval_service.py      # 评测服务（问题驱动 + IoU/Top-Hit 计算）
├── run_mevid_eval.py    # 端到端评测驱动脚本
├── 接口文档.md           # 本文件
└── data/
    └── runtime/
        └── {namespace}/
            ├── episodic_events.sqlite
            ├── chroma/
            └── fastapi_eval_result.json
```
