## 实验 1 vs 2 对比

| 指标 | SQL ON (1) | SQL OFF (2) | Δ | 方向 |
|------|------------|-------------|-----|------|
| TopHit | 0.7333 | 0.7333 | 0 | — |
| Precision | 0.4735 | 0.4694 | -0.0041 | ≈ |
| **Recall** | 0.5333 | **0.6000** | **+0.0667** | ↑ |
| Factual | 0.5667 | 0.5667 | 0 | — |
| CustomCorr | 0.6296 | 0.6296 | 0 | — |
| **E2E** | 0.5779 | **0.6088** | **+0.0309** | ↑ |
| **IoU** | 0.4572 | **0.4172** | **-0.0400** | ↓ |
| VidMatch | 0.64 | 0.64 | 0 | — |
| Latency | 13.0 | 12.2 | -0.8s | ↑ |

### 分析

- **Recall +6.7%**：纯 hybrid 检索的语义覆盖面更广，SQL 的结构化过滤在 RRF 融合时限制了召回
- **E2E +3.1%**：综合评分提升，主要来自 Recall 改善
- **IoU -4.0%**：SQL 的确定性过滤对时间定位有帮助（结构化字段如 `start_time`）
- **Precision 持平**：虽然 easy 精度大涨（0.72→0.89），但 medium 拖了后腿（0.42→0.26），整体打平

### 难度分层对比

| | easy Δ | medium Δ | hard Δ |
|------|--------|----------|--------|
| Precision | +0.1696 | -0.1583 | 0 |
| Recall | +0.0500 | 0 | +0.1500 |
| E2E | +0.1099 | -0.0942 | +0.0683 |

- **Easy 大幅受益**：SQL 的噪声被移除后，简单问题的 precision 达到 0.89
- **Medium 变差**：中等难度问题依赖 SQL 的结构化线索（object_type/color/zone）
- **Hard 改善有限**：困难问题的瓶颈不在分支选择，而在检索本身

## 操作速查

```bash
cd agent/test

# 消融：关 SQL
python run_chunks.py --max-chunks 1 --disable-sql

# 消融：换模型
AGENT_RERANK_MODEL=cross-encoder/ms-marco-MiniLM-L-6-v2 python run_chunks.py --max-chunks 1

# 消融：换融合模式（待实现 --fusion-mode）
AGENT_FUSION_MODE=concat_dedupe python run_chunks.py --max-chunks 1

# 看结果
cat generated/ragas_eval_p4_chunks/results.csv

# 分难度
python3 << 'EOF'
import json
from pathlib import Path
chunk = Path("generated/ragas_eval_p4_chunks/chunk01")
norm = json.loads((chunk / "dataset_part1_part4/agent_test_normalized.json").read_text())
diff = {c["case_id"]: c["difficulty_level"] for c in norm}
e2e = json.loads((chunk / "e2e_report.json").read_text())
for level in ["easy","medium","hard"]:
    cases = [c for c in e2e["cases"] if diff.get(c["case_id"]) == level]
    n = len(cases); p = sum((c.get("ragas",{}).get("retrieval",{})or{}).get("context_precision",0)or 0 for c in cases)/n
    r = sum((c.get("ragas",{}).get("retrieval",{})or{}).get("context_recall",0)or 0 for c in cases)/n
    e = sum((c.get("ragas",{}).get("end_to_end",{})or{}).get("ragas_e2e_score",0)or 0 for c in cases)/n
    i = sum((c.get("temporal",{})or{}).get("temporal_iou",0)or 0 for c in cases)/n
    print(f"{level:<8} n={n} prec={p:.4f} rec={r:.4f} e2e={e:.4f} iou={i:.4f}")
EOF
```
