## Top-Hit 未命中 & IoU 分析

**Q06（cross_camera，Top-Hit ❌）**：白色衬衫人物（person_global_1）G328→G424，检索 Top-1 返回 G328 而非 G424，但 agent 通过多路融合仍正确作答。

**Q07（temporal，IoU=0.000）**：Agent 正确回答 yes，但从回答中提取的时间窗（约 0:54）落在 G421 另一人（person_global_3，深色外套）的时间段，而非目标人物 person_global_4（深色夹克格子衬衫，2:08–3:44）。答案正确，时间定位发生混淆——G421 中两个深色服装的人外观相近，时间约束描述对 agent 的时间提取能力是挑战。

## RAG Pipeline 配置说明

| 配置项 | 当前值 | 说明 |
|--------|--------|------|
| 检索层级 | child（track 级） | event 级切换后 IoU 无变化 |
| Parent Projection | 关闭 | 开启后时间窗扩展为 10 分钟，IoU 反而下降 |
| 执行模式 | parallel_fusion | SQL + Hybrid 并行融合 |
| Chroma 集合 | child + global_entity | GE 集合用于跨摄像头 verifier |

**IoU 说明**：Mean IoU 0.376（±30s padding）；temporal 类问题 IoU 最高（0.457），得益于时间约束缩小了检索范围。cross_camera 类 IoU 0.362 主要受限于 GE 集合中时间范围为聚合值（非精确 clip 时间）。
