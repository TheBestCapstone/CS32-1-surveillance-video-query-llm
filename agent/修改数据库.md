# 修改数据库（快速同步技能）

## 场景
- 当 `events_vector_flat` 原字段发生变化
- 当 SQL 检索字段与向量检索字段职责需要调整
- 当需要新增/删除索引或字段时

## 设计原则（当前版本）
- SQL 检索只负责结构化/范围过滤字段（如 object/color/zone/time/bbox）
- 向量数据库负责语义召回（event_text/summary/keywords 等）
- 两者通过同一 `event_id / vector_ref_id` 互补，不重复承担职责

## 一次修改要改哪些文件
- 字段与索引定义（单一真源）：
  - [schema.py](file:///home/yangxp/Capstone/agent/db/schema.py)
- seed 映射与写入逻辑：
  - [sqlite_builder.py](file:///home/yangxp/Capstone/agent/db/sqlite_builder.py)
- 如有旧字段引用，修复工具层：
  - [sql_tools.py](file:///home/yangxp/Capstone/agent/tools/sql_tools.py)
  - [hybrid_tools.py](file:///home/yangxp/Capstone/agent/tools/hybrid_tools.py)
  - [types.py](file:///home/yangxp/Capstone/agent/node/types.py)

## 快速同步步骤
1. 修改 [schema.py](file:///home/yangxp/Capstone/agent/db/schema.py)
- 更新 `CREATE_TABLE_SQL`
- 更新 `INDEX_SQL_LIST`
- 更新 `INSERT_COLUMNS`

2. 修改 [sqlite_builder.py](file:///home/yangxp/Capstone/agent/db/sqlite_builder.py)
- 在 `_load_seed_rows()` 里更新 seed 字段映射
- 确保 `vector_doc_text/vector_ref_id` 仍可稳定生成

3. 重建数据库并验证
```bash
python -m agent.db.manage_graph_db build \
  --db-path /home/yangxp/Capstone/data/SQLite/episodic_events.sqlite \
  --seed-json /home/yangxp/Capstone/data/basketball_output/basketball_1_events_vector_flat.json \
              /home/yangxp/Capstone/data/basketball_output/basketball_2_events_vector_flat.json \
  --reset
```

4. 检查表与索引
```bash
python - << 'PY'

p='/home/yangxp/Capstone/data/SQLite/episodic_events.sqlite'
conn=sqlite3.connect(p); cur=conn.cursor()

print('columns=', [x[1] for x in cur.fetchall()])

conn.close()
PY
```
