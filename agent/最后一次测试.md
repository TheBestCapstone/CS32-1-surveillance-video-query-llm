## 本次改动总结

### P0 修复

- `extract_structured_filters`: 词边界匹配，消除子串误匹配（`scar`→`car`、`personal`→`person`）
- FTS5 注释修正 + 空 token 防御

### P1 修复

- 多实体查询不再过度过滤（仅 1 个实体时设 hard filter）
- `_run_sql_branch` 集成 `build_text2sql_plan` + FTS5 + relaxed fallback
- 启用同义词扩展（`vehicle`→`car`、`elderly`→`person`）、词干提取

### P2 修复

- `execute_dynamic_sql`: 自动 LIMIT 200、查询超时 30s、WAL+只读模式
- 停用词统一：3 处分散定义合并为 `SQL_TOKEN_STOPWORDS`（119 词）

### P3 修复

- Token 硬截断改为可配置 `AGENT_SQL_TOKEN_LIMIT`（默认 10）
- JSON 解析增强：`[SQL_JSON_START]`/`[SQL_JSON_END]` 标记 + legacy fallback + 日志
- SQLite 连接优化：WAL、cache_size=-8000、synchronous=NORMAL

### Prompt 优化（6 个）

- Self-query: 增加 expansion 好/坏示例，明确输出 schema
- Query Classifier: 每个 label 增加具体示例
- Summary Node: 合并散落格式指令，修复注释混入 prompt 的 bug
- Match Verifier: 详细 exact/partial/mismatch 定义
- Pure SQL / Hybrid sub-agent: 精简工作流描述
