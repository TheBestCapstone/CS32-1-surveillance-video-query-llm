# 消融实验记录（三种检索形态）

> 分支：`cursor/消融实验`  
> 评测集：qwen3.5-max **chunk01**（10 视频 × 3 = **30 case**）  
> 基线 Reranker：`jinaai/jina-reranker-v2-base-multilingual`，Chroma **event** 级（与当前 qwen3.5-max 跑分一致）

本节把检索通路抽象为三类：**仅向量**、**仅 SQL**、**向量 + SQL 双开**。  
**双开**一行与仓库内 `[测试结果汇总.md](测试结果汇总.md)` **qwen3.5-max 基线 · Chunk01** 对齐（30 cases）；**仅向量 / 仅 SQL** 为同评测口径下的消融对照。

---

## 三种形态定义


| 形态             | 含义                                                  | 典型开关 / 条件                                       |
| -------------- | --------------------------------------------------- | ----------------------------------------------- |
| **仅向量**        | 关闭 SQL 分支，检索走 hybrid（向量/Chroma 等），无结构化 SQL 候选参与 RRF | `AGENT_DISABLE_SQL_BRANCH=1` 或 `--disable-sql`  |
| **仅 SQL**      | 仅结构化检索通路可用；语义向量分支不参与融合                              | SQL-only / fallback                             |
| **双开（SQL ON）** | SQL 与向量两路并行，经 RRF 等融合                               | 默认：SQL 分支 ON + `AGENT_BUILD_VIDEO_COLLECTION=1` |

## 两两对比要点

### 1. 双开 vs 仅向量（SQL ON vs OFF）


| 指标         | 双开        | 仅向量       | Δ（仅向量 − 双开） |
| ---------- | --------- | --------- | ----------- |
| TopHit     | 0.800     | 0.787     | −0.013      |
| Precision  | 0.502     | **0.518** | **+0.016**  |
| Recall     | 0.717     | **0.738** | **+0.021**  |
| Factual    | **0.633** | 0.608     | −0.025      |
| CustomCorr | 0.623     | 0.629     | +0.006      |
| E2E        | 0.626     | **0.651** | **+0.025**  |
| IoU        | **0.417** | 0.358     | **−0.059**  |
| VidMatch   | **0.640** | 0.622     | −0.018      |
| Latency    | 11.7s     | 11.2s     | −0.5s       |


**解读：**

- **仅向量在检索召回与端到端更占优**：Recall、Precision、E2E 略优于双开，语义通路补充了 SQL 难以覆盖的表述变体。
- **双开在事实一致性、视频对齐上更稳**：Factual、VidMatch 更高；双开相比仅向量 **IoU 更高**（0.417 vs 0.358），说明 SQL 分支仍有助于把时间窗对齐到结构化时段。
- **IoU、CustomCorr、Precision 的峰值在纯 SQL**（见下一小节）；双开主要折中 **TopHit / Recall / E2E**。

### 2. 双开 vs 仅 SQL


| 指标         | 仅 SQL     | 双开        | 趋势          |
| ---------- | --------- | --------- | ----------- |
| TopHit     | 0.733     | **0.800** | 双开 ↑        |
| Precision  | 0.471     | **0.502** | 双开 ↑        |
| Recall     | 0.545     | **0.717** | 双开 ↑        |
| Factual    | 0.588     | **0.633** | 双开 ↑        |
| CustomCorr | **0.658** | 0.623     | **仅 SQL ↑** |
| E2E        | 0.571     | **0.626** | 双开 ↑        |
| IoU        | **0.478** | 0.417     | **仅 SQL ↑** |
| VidMatch   | 0.652     | 0.640     | 仅 SQL ↑     |
| Latency    | **10.2**  | 11.7      | 仅 SQL 更快    |


**解读：**

- **向量分支拉高 TopHit / Recall / E2E / Factual**：双开在「找得到、答得全」上优于纯 SQL。
- **纯 SQL 仍守住多块长板**：**IoU**、**CustomCorr** 与 **Precision** 最高——确定性时间戳与结构化字段过滤，减少跨片段漂移；自定义评分更贴合规则型期望答案。
- **工程含义**：若评测权重极度看重 **时段对齐** 或 **自定义正确率**，可增加 SQL 侧权重或单独保留 SQL-only 作对照；综合体验仍以 **双开** 为主路径。

### 3. 仅向量 vs 仅 SQL

- **E2E / Recall**：仅向量（0.651 / 0.738）高于仅 SQL（0.571 / 0.545），语义检索不可替代。
- **IoU / CustomCorr**：仅 SQL（0.478 / 0.658）高于仅向量（0.358 / 0.629），结构化时间在场景约束题上更占优。
- **结论**：二者互补；**双开**在多数 headline 指标上折中，并与 `测试结果汇总.md` Chunk01 公开发布数字一致。

## 操作速查

```bash
cd agent/test

# 仅向量：关 SQL
python run_chunks.py --max-chunks 1 --disable-sql

# 双开：默认（勿设 AGENT_DISABLE_SQL_BRANCH）
AGENT_RERANK_MODEL=jinaai/jina-reranker-v2-base-multilingual \
AGENT_BUILD_VIDEO_COLLECTION=1 \
python run_chunks.py --max-chunks 1

# 看结果
cat generated/ragas_eval_p4_chunks/results.csv

# 分难度
python3 << 'EOF'
import json
from pathlib import Path
chunk = Path("generated/ragas_eval_p4_chunks/chunk01")
norm = json.loads((chunk / "dataset_part1_part4/agent_test_normalized.json").read_text())
diff = {c["case_id"]: c["difficulty_level"] for c in norm}
e2e = json.loads((chunk / "e2e_report.json").read_text())
for level in ["easy","medium","hard"]:
    cases = [c for c in e2e["cases"] if diff.get(c["case_id"]) == level]
    n = len(cases); p = sum((c.get("ragas",{}).get("retrieval",{})or{}).get("context_precision",0)or 0 for c in cases)/n
    r = sum((c.get("ragas",{}).get("retrieval",{})or{}).get("context_recall",0)or 0 for c in cases)/n
    e = sum((c.get("ragas",{}).get("end_to_end",{})or{}).get("ragas_e2e_score",0)or 0 for c in cases)/n
    i = sum((c.get("temporal",{})or{}).get("temporal_iou",0)or 0 for c in cases)/n
    print(f"{level:<8} n={n} prec={p:.4f} rec={r:.4f} e2e={e:.4f} iou={i:.4f}")
EOF
```
