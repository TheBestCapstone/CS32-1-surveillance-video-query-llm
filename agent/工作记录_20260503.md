### 下午 — Tier 1 & 2 设计与实施

**Tier 1：Video Collection + two-stage retrieval**


| 文件                                       | 内容                                                            |
| ---------------------------------------- | ------------------------------------------------------------- |
| `tools/video_discriminator.py`           | LLM（qwen3-max）生成 per-video 区分度 summary                        |
| `db/chroma_builder.py`                   | `_build_video_records()` + video_collection 集成                |
| `db/config.py`                           | `get_graph_chroma_video_collection()`                         |
| `tools/hybrid_tools.py`                  | `video_filter` → Chroma `$in` where                           |
| `node/parallel_retrieval_fusion_node.py` | `_coarse_video_filter()` 嵌入 `_run_hybrid_branch`              |
| `test/ragas_eval_runner.py`              | `_build_discriminator_llm()` + `AGENT_BUILD_VIDEO_COLLECTION` |


关键设计决策：

- coarse filter 嵌入 hybrid branch 内部（非独立线程），embedding 经 P1-3 LRU 复用
- video collection 名从 child collection 自动派生
- latency overhead ~0.4s（27-case 验证）

**Tier 2：Zero-LLM scene attribute filtering**


| 文件                                       | 内容                                                               |
| ---------------------------------------- | ---------------------------------------------------------------- |
| `tools/scene_attrs.py`                   | SQL 提取 `object_type`/`scene_zone`/`object_color` → `has_X` + IDF |
| `node/self_query_node.py`                | `scene_constraints` schema + vocab 注入 LLM prompt                 |
| `node/parallel_retrieval_fusion_node.py` | `_apply_scene_boost()` RRF 后重排（λ=0.1）                            |
| `lightingRL/prompt_registry.py`          | system prompt 加 scene vocab 指令                                   |
| `test/ragas_eval_runner.py`              | build pipeline：SQLite 建库后自动 `build_scene_attrs()`                |


关键设计决策：

- 词表从 SQL 自动生成，零 LLM、零手工、每次 `--prepare-subset-db` 自动更新
- Boost 非 filter（默认），仅 query_weight≥0.95 且硬约束走 filter
- IDF = `log(N/df) / log(N)`，与 query_weight 解耦
- 颜色全部纳入，高频属性靠 IDF 压低权重
- self_query LLM 只做选择（从词表中选），不生成新属性
