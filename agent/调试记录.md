## 2026-05-02 P1-3 query embedding LRU + 磁盘缓存

### 1. 决策记录

### 2. 现状

`agent/tools/llm.py:58 get_qwen_embedding` 是 embedding 单一入口，被 6 个文件调用：

### 3. 实施

完整方案与代码参见 `agent/todo.md` "P1-3 query embedding LRU + 磁盘缓存" 段落（含 env knobs / 单测覆盖 / 烟测结果）。

技术亮点：
- 自实现轻量 LRU（`OrderedDict` + `threading.Lock`），可"只查不发"以支持批量路径正确合并 cache miss 成单次 batch API call
- 磁盘缓存按 `sha256(provider|model|dimensions|text)` 做 key，写入用 `tempfile + rename` 防 partial write
- 单条 / 批量都走 `_lookup_cached` 统一查询路径，两层 cache 一致性强
- 批量路径里 cache miss 项按 `EMBEDDING_BATCH_LIMIT=10` 分批一次性 batch API call → 比"逐项 cache 查"少了大量 round-trip

- `AGENT_EMBEDDING_CACHE_LRU_SIZE` 默认 2048，对 50-case eval 充裕（2048 远大于 50）；若遇大 corpus 建库批量，可临时调到 8192+
- 单测里 floor 改成 1（原来 64），允许小容量 eviction 场景测试
- LRU 的 hit 计数用 `_bump("lru_hits")` 直接写 stats dict，**不要**用 `cache_info().hits` —— OrderedDict 实现没那玩意
