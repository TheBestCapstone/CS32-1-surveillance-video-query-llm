# 测试模块汇总（Agent 能力落地验证）

## 一、主流程编排层

### 1) `graph_builder.py` / `graph.py`
- 核心功能点：
- 组装主图执行模式（`parallel_fusion` 与 `legacy_router`）。
- 连接关键节点：分类、并行检索融合、最终回答。
- 测试重点：
- 模式切换正确性（环境变量驱动）。
- 节点拓扑完整性与可达性（无死链、无不可达链路）。
- 关键状态字段在跨节点过程中的传递完整性。

### 2) `node/types.py`
- 核心功能点：
- 定义 `AgentState` 契约、输入提取、状态重置、query 清洗与结构转换。
- 测试重点：
- `extract_latest_query` 在多消息场景下稳定抽取最后一条用户输入。
- `reset_ephemeral_state` 对临时字段的清理完整性。
- `question_to_meta_and_event` 的结构映射一致性。

## 二、路由与决策层

### 3) `node/query_classification_node.py` + `agents/shared/query_classifier.py`
- 核心功能点：
- 使用 `LLM + structured output` 完成 `structured/semantic/mixed` 分类。
- 输出兼容模式到 `tool_choice`，用于并行融合权重偏置。
- 测试重点：
- 结构化输出解析正确性与异常兜底正确性。
- 分类标签与 `tool_choice.mode` 映射一致性。
- 同一查询多次执行的稳定性（低抖动）。

### 4) `node/tool_router_node.py` + `node/router_prompts.py`（legacy 路由链）
- 核心功能点：
- 四元组提取与路由决策均由 `LLM + schema` 输出。
- 反思重试场景下可复用优化条件继续决策。
- 测试重点：
- LLM 输出不合法时 fallback 行为可用且安全。
- `route_reason_codes`、`route_confidence` 写回完整。
- 重试路径（reflection -> router）不中断、无状态污染。

### 5) `node/reflection_node.py`
- 核心功能点：
- 对结果质量进行审查、错误定位、策略生成、重试控制与终止决策。
- 测试重点：
- `needs_retry/can_continue/max_retries_reached` 状态机正确性。
- 异常分支、校验失败分支、最大重试分支可控收敛。
- 反思结果与主图路由协同一致性。

## 三、执行与检索层

### 6) `node/parallel_retrieval_fusion_node.py`
- 核心功能点：
- 并行执行 SQL 与 Hybrid 分支，进行 Weighted RRF 融合。
- 支持分支超时/失败降级及双失败兜底。
- 测试重点：
- 双分支并行时延与超时行为。
- 单分支失败时降级输出正确性。
- 融合结果可解释元信息（`fusion_meta`）完整性。

### 7) `node/pure_sql_node.py` + `agents/pure_sql/sub_agent.py`
- 核心功能点：
- 结构化查询规划与 SQL 工具调用。
- 测试重点：
- 过滤条件命中率（object/color/zone/time）与误召回控制。
- SQL 异常与空结果处理。

### 8) `node/hybrid_search_node.py` + `agents/hybrid_search/sub_agent.py`
- 核心功能点：
- 语义检索工具调用、候选召回与结果整形。
- 测试重点：
- 向量检索返回质量、字段完整性、空召回处理。
- 与 SQL 分支融合前的结果契约一致性。

## 四、工具与数据访问层

### 9) `tools/sql_tools.py` / `tools/db_access.py`
- 核心功能点：
- schema 探测、动态 SQL 执行、SQLite/LanceDB/Chroma 访问封装。
- 测试重点：
- SQL 安全性（非法字段、空字段、注入风险字符串）。
- 数据源切换与路径配置正确性。

### 10) `tools/hybrid_tools.py`
- 核心功能点：
- 语义检索入口、动态权重查询工具封装。
- 测试重点：
- embeddings 异常、远端服务错误、空输入容错。
- 结果字段标准化兼容（供融合层消费）。

### 11) `agents/shared/fusion_engine.py`
- 核心功能点：
- Weighted RRF 融合权重装载与排名融合计算。
- 测试重点：
- `structured/semantic/mixed` 权重是否正确生效。
- 分支输入为空或重复主键时结果稳定性。

## 五、结果产出层

### 12) `node/answer_node.py`
- 核心功能点：
- 将融合结果转为最终答复文本与状态字段。
- 测试重点：
- 正常结果、空结果、工具错误场景下文案正确性。
- 关键字段（`final_answer/tool_error/search_explain`）一致性。

### 13) `test/result_test_runner.py`
- 核心功能点：
- 驱动 case 执行并生成 `result.md` / `result_report.json`。
- 测试重点：
- 断言项覆盖完整度（route/tool_error/min_results）。
- 报告可追踪性（每 case 时延、失败类别、top5）。

## 六、优先级建议
- `P0`：`query_classifier`、`parallel_retrieval_fusion_node`、`answer_node`、`result_test_runner`。
- `P1`：`tool_router_node`、`reflection_node`、`types` 状态管理。
- `P2`：`retrieval/event_retriever.py`、`db/manage_graph_db.py` 等旁路与运维模块。
