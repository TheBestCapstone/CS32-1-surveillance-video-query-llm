# 测试要求规格说明书（多维验证）

## 1. 测试目标与范围
- 目标：验证已配置 Agent 能力是否在真实篮球数据上“可执行、可解释、可回归、可退化”。
- 范围：主图链路、路由分类、并行检索融合、结果产出、异常降级、性能基线。
- 非目标：模型训练质量评估、UI 交互测试。

## 2. 总体验收门槛
- 功能通过率：`P0` 用例必须 `100%`，全量用例 `>= 95%`。
- 稳定性：同一用例重复 10 次，关键断言波动不超过 `10%`。
- 性能：核心链路 p95 不高于基线的 `1.5x`（环境波动可放宽到 `2x`）。
- 鲁棒性：异常场景不崩溃，必须输出可解释 `tool_error` 或 fallback 结果。

## 3. 维度一：功能测试（Functional）

### 3.1 路由分类能力
- 覆盖项：
- `structured/semantic/mixed` 标签判定。
- 标签到 `tool_choice` 兼容映射正确性。
- `routing_metrics` 字段完整写回。
- 关键断言：
- `classification_result.label` 在合法集合内。
- `tool_choice.mode` 与期望集合一致。

### 3.2 并行检索与融合能力
- 覆盖项：
- SQL/Hybrid 分支均执行。
- Weighted RRF 按 label 权重生效。
- 融合输出排序与去重有效。
- 关键断言：
- `sql_result/hybrid_result/merged_result` 结构合法。
- `fusion_meta.method == weighted_rrf`。

### 3.3 最终回答能力
- 覆盖项：
- 正常召回、空召回、单分支降级、双分支失败。
- 关键断言：
- `final_answer` 非空且与结果集一致。
- 出错时 `tool_error` 与答复语义一致，不互相冲突。

## 4. 维度二：性能测试（Performance）

### 4.1 指标定义
- `end_to_end_ms`：单 query 从输入到 `final_answer` 耗时。
- `sql_branch_ms` / `hybrid_branch_ms`：分支耗时。
- `fusion_ms`：融合耗时。
- `p50/p95`：按场景组分别统计。

### 4.2 性能要求
- `pure_sql` 主导场景：p95 参考 `perf_baseline.sql_p95_ms`，允许 `<= 1.5x~2x`。
- `hybrid` 主导场景：p95 参考 `perf_baseline.hybrid_p95_ms`，允许 `<= 1.5x~2x`。
- 并行链路：不得出现串行退化（总耗时接近两分支求和需告警）。

## 5. 维度三：异常处理测试（Resilience）

### 5.1 输入异常
- 空 query、超长 query、噪声字符 query、混合语言 query。
- 期望：不崩溃，返回可解释 fallback。

### 5.2 外部依赖异常
- LLM 返回非 JSON/字段缺失。
- Hybrid 检索上游报错或超时。
- SQLite 查询失败（路径错误/锁冲突）。
- 期望：触发 fallback 或降级，不中断主流程。

### 5.3 状态与重试异常
- `retry_count` 接近上限。
- `reflection_result` 字段缺失或格式异常。
- 期望：重试机制受控，能收敛到终态。

## 6. 维度四：边界条件测试（Boundary）

### 6.1 结果数量边界
- `0` 条、`1` 条、大量结果（上限截断）。
- 期望：回答层描述与数量一致，无越界访问。

### 6.2 字段边界
- `object_color_en=unknown`、稀有 `scene_zone_en`、长 `event_text_en`。
- 期望：过滤与展示稳定，空值不导致异常。

### 6.3 路由边界
- 结构化与语义混合提问。
- 模糊意图、极短 query。
- 期望：标签合法，分支权重可解释。

## 7. 测试用例分层策略
- `L0` 冒烟：5~10 条核心路径，提交前必跑。
- `L1` 回归：按能力域扩展到 40~80 条。
- `L2` 压测/鲁棒：批量随机 query 与故障注入。

## 8. 报告规范（输出到 test/）
- 必含文件：
- `test_report_summary.md`：总体通过率、性能统计、失败分类。
- `test_report_detail.json`：逐 case 输入、输出、断言、耗时、错误详情。
- `test_report_trends.md`：与历史基线对比（路由准确率、召回率、p95）。
- 必含字段：
- `case_id`、`query`、`expected`、`actual`、`status`、`elapsed_ms`、`failure_category`。

## 9. 风险与控制
- 数据偏态（几乎全是 person）会放大“伪通过”风险。
- 建议加入对抗样本与负样本增强（不存在对象、矛盾条件、跨域词汇）。
- 混合检索依赖外部 embedding 服务，需独立监控失败率和超时率。
